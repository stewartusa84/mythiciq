# MythicIQ Performance Signals for Future Development

This is a working list of player performance signals MythicIQ may use for badges, queue eligibility, clean-run evaluation, group-fit scoring, and future role-based feedback. These are not all intended to be public in every queue. Visibility should depend on run intent.

## General Run Performance

* Completed runs
* Timed runs
* Clean runs
* Timed clean runs
* Recent successful runs in the same key band
* Recent successful runs in the same dungeon
* Key-band experience
* Role played during the run
* Spec played during the run
* Death count
* Avoidable deaths
* Killing blows taken
* Large damage taken
* Repeated avoidable damage patterns
* Curated mechanic failures
* Curated mechanic successes
* Infraction count against active ruleset
* Evidence confidence level

## Damage / DPS Signals

* Overall DPS
* Boss DPS
* Trash DPS
* Priority target damage
* Damage during required burn windows
* Death-adjusted DPS
* Downtime / inactivity estimate
* Role-adjusted DPS threshold
* Key-band-adjusted DPS threshold
* Damage while maintaining acceptable mechanics
* Damage contribution without excessive avoidable damage

## Healing / HPS Signals

## Badge / Visibility Philosophy

Badges should generally represent demonstrated baseline competence or positive behavior, not public tier rankings. MythicIQ should avoid public labels such as “A+ DPS,” “S-Tier DPS,” or “Flawless-only player.”

Badge visibility should depend on the queue:

* Growth and learning queues should show only basic trust, intent, and cooperative-play signals.
* Crest, vault, and timing queues should show relevant baseline competency signals such as verified character, interrupt awareness, recent completions, and clean/timed-clean experience.
* Progression push queues may show stronger readiness signals such as consumable usage, enhancement status, DPS readiness, mitigation readiness, healer cooldown readiness, and recent push experience.

The goal is to align the visible signals with the social contract of the run, not to give players another tool for unnecessary exclusion.
